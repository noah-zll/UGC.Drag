# UGC.Drag 拖拽组件设计文档

## 1. 组件简介

UGC.Drag 是一个基于 Unity UGUI（EventSystem + GraphicRaycaster）的通用拖拽/放置（Drag & Drop）组件包，提供可复用的拖拽源（Drag Source）与投放目标（Drop Target）能力，并通过统一的拖拽上下文（Drag Context）与事件回调，支持常见的 UI 拖拽交互需求（背包、装备槽、编辑器面板、列表重排等）。

设计目标：

- 在不侵入业务的前提下，提供稳定、可扩展的拖拽管线
- 默认实现覆盖 80% 场景，复杂场景通过接口扩展
- 兼容不同 Canvas 渲染模式与多分辨率/多 DPI

## 2. 功能特性

- **完整拖拽流程**：BeginDrag / Drag / EndDrag 全链路封装
- **拖拽视觉**：支持拖拽“影子/替身”跟随指针移动，可替换生成策略
- **目标命中**：拖拽期间 Raycast 命中 DropTarget，支持 Hover Enter/Exit
- **接收判定**：目标可按 payload、来源、规则决定是否可接收
- **结果回传**：Drop 成功/失败/取消统一结果对象回调给 Source
- **可扩展**：为重排、交换、合并、跨容器移动预留接口
- **不阻挡射线**：拖拽视觉默认不阻挡 UI 射线，确保命中准确

## 3. 技术架构

### 3.1 核心组件

- **DragService**：拖拽调度器（单例或场景节点），维护当前拖拽会话、执行 Raycast、驱动视觉与 Hover 状态机
- **DragSource**：拖拽源组件，提供 payload，响应拖拽结果
- **DropTarget**：投放目标组件，提供可接收判定与 Drop 处理逻辑
- **DragVisual**：拖拽视觉表现，负责跟随指针并在结束时销毁/回收

### 3.2 数据模型

- **DragContext**：拖拽上下文（源、payload、PointerEventData、当前 Hover 目标等）
- **DragResult**：拖拽结果（Success、Target、Reason 等）

### 3.3 流程概览

```
DragSource(OnBeginDrag)
        |
        v
DragService.Begin(context)  --> Create DragVisual
        |
        v
DragService.Tick/OnDrag     --> Raycast --> Hover Target (Enter/Exit)
        |
        v
DragSource(OnEndDrag)
        |
        v
DragService.End(context)    --> If target.CanAccept then target.OnDrop
        |
        v
DragSource.OnDragEnded(context, result) --> 成功/失败/回滚
```

## 4. API 设计（拟定）

为与现有 UGC.* 包保持一致，建议命名空间与 asmdef rootNamespace 采用 `UGC.Drag`，并按 Core/Interfaces/Models 划分。

### 4.1 Models

```csharp
namespace UGC.Drag.Models
{
    public sealed class DragContext
    {
        public object Payload { get; }
        public UGC.Drag.Runtime.DragSource Source { get; }
        public UGC.Drag.Runtime.DropTarget CurrentHover { get; internal set; }
        public UnityEngine.EventSystems.PointerEventData EventData { get; }
    }

    public readonly struct DragResult
    {
        public bool Success { get; }
        public UGC.Drag.Runtime.DropTarget Target { get; }
        public string Reason { get; }
    }
}
```

### 4.2 Interfaces

```csharp
namespace UGC.Drag.Interfaces
{
    public interface IDragSource
    {
        bool CanBeginDrag(UnityEngine.EventSystems.PointerEventData eventData);
        object BuildPayload();
        void OnDragStarted(UGC.Drag.Models.DragContext context);
        void OnDragEnded(UGC.Drag.Models.DragContext context, UGC.Drag.Models.DragResult result);
    }

    public interface IDropTarget
    {
        bool CanAccept(UGC.Drag.Models.DragContext context);
        void OnDragEnter(UGC.Drag.Models.DragContext context);
        void OnDragExit(UGC.Drag.Models.DragContext context);
        void OnDrop(UGC.Drag.Models.DragContext context);
    }

    public interface IDragVisualFactory
    {
        UnityEngine.RectTransform CreateVisual(
            UGC.Drag.Models.DragContext context,
            UnityEngine.Transform visualRoot);
    }
}
```

### 4.3 Runtime 组件（拟定）

```csharp
namespace UGC.Drag.Runtime
{
    public sealed class DragService : UnityEngine.MonoBehaviour
    {
        public UnityEngine.Transform visualRoot;
        public UnityEngine.UI.GraphicRaycaster raycaster;

        public bool IsDragging { get; }
    }

    public sealed class DragSource : UnityEngine.MonoBehaviour,
        UnityEngine.EventSystems.IBeginDragHandler,
        UnityEngine.EventSystems.IDragHandler,
        UnityEngine.EventSystems.IEndDragHandler
    {
        public DragService service;

        public event System.Action<UGC.Drag.Models.DragContext> onDragStarted;
        public event System.Action<UGC.Drag.Models.DragContext, UGC.Drag.Models.DragResult> onDragEnded;
    }

    public sealed class DropTarget : UnityEngine.MonoBehaviour
    {
        public event System.Action<UGC.Drag.Models.DragContext> onDrop;
    }
}
```

## 5. 使用流程

### 5.1 基本使用

1. 场景中确保存在 `EventSystem`
2. Canvas 上确保存在 `GraphicRaycaster`
3. 在 UI 根节点挂载 `DragService`，配置 `raycaster` 与 `visualRoot`
4. 在可拖拽 UI 节点挂载 `DragSource`，绑定 `service`
5. 在可接收区域挂载 `DropTarget`，实现/绑定 `IDropTarget` 处理逻辑

### 5.2 代码示例（目标接收）

```csharp
public sealed class InventorySlotTarget : UnityEngine.MonoBehaviour, UGC.Drag.Interfaces.IDropTarget
{
    public bool CanAccept(UGC.Drag.Models.DragContext context)
        => context.Payload is ItemData;

    public void OnDragEnter(UGC.Drag.Models.DragContext context) { }
    public void OnDragExit(UGC.Drag.Models.DragContext context) { }

    public void OnDrop(UGC.Drag.Models.DragContext context)
    {
        var item = (ItemData)context.Payload;
        // 放入格子逻辑
    }
}
```

## 7. 性能优化

- Raycast 仅在拖拽期间执行
- Hover 目标缓存，仅在变化时触发 Enter/Exit
- DragVisual 默认 `blocksRaycasts = false`，避免遮挡目标命中
- 视觉对象可选用对象池以减少 GC

## 8. 注意事项

- 多 Canvas 或多 Raycaster 场景需明确 DragService 选用的 Raycaster
- World Space Canvas 需要正确相机参与坐标换算（eventData.pressEventCamera）
- 若需要“拖拽把自己移动到视觉根节点”，建议用 DragVisual 影子而不是直接移动源物体

## 9. 兼容性要求

- Unity 2020.3 或更高版本
- 使用 Unity UGUI（UnityEngine.UI）与 EventSystem
